# 2025년 2학기 프로젝트  
## 온라인 사용자 리뷰 기반 감성·요구사항 통합 분석 시스템

본 프로젝트는 2025년도 2학기 동안 수행한 연구/개발 프로젝트로,  
온라인 사용자 리뷰 데이터를 활용하여 감성(Sentiment)과 요구사항(Requirement)을 동시에 분석하는  
딥러닝 기반 통합 분석 모델을 설계·구현하는 것을 목표로 한다.

---

## 📌 프로젝트 개요

- 프로젝트 기간: 2025년 9월 ~ 2025년 12월  
- 진행 형태: 개인 프로젝트  
- 개발 환경: Python, Google Colab, TensorFlow(Keras)

---

## 🎯 프로젝트 목표

- 온라인 리뷰 텍스트에서  
  - 감성 표현(긍·부정)
  - 사용자 요구사항(불만, 개선 요구)
  를 하나의 모델에서 동시에 예측
- 기존 단일 출력 모델의 한계를 극복하고
  - 오류 전이(Error Propagation) 문제 감소
  - 모델 효율성 및 실용성 향상
- 실제 서비스에 적용 가능한 분석 파이프라인 제시

---

## 🧠 시스템 구조

본 프로젝트에서는 다음과 같은 구조를 사용하였다.

1. 리뷰 데이터 수집 및 전처리
2. 형태소 분석 및 토큰화
3. Word2Vec 기반 임베딩
4. LSTM 기반 공통 은닉층 학습
5. 이중 출력층(Dual Output Layer)
   - 감성 분류 출력
   - 요구사항 분류 출력

---

## 🗂 데이터셋

- 데이터 출처: 온라인 배달 앱 사용자 리뷰 (예: 요기요)
- 데이터 규모: 30,000건
- 라벨 구성
  - 감성 라벨: 긍정 / 부정
  - 요구사항 라벨: 요구 있음 / 요구 없음
- 불균형 문제 해결
  - SMOTE 기반 오버샘플링 적용

---

## ⚙️ 개발 환경 및 라이브러리

- Language: Python 3.2.2  
- Framework: TensorFlow 2.1 (Keras API)  
- Libraries
  - NumPy, Pandas
  - scikit-learn
  - gensim (Word2Vec)
  - imbalanced-learn (SMOTE)

---

## 📊 실험 결과 요약

- Dual-Output LSTM 모델은
  - 단일 출력 모델 대비
  - 감성 분석과 요구사항 분석 모두에서 F1-score 향상
- 특히 요구사항(소수 클래스) 탐지 성능이 유의미하게 개선됨

---

## 🔍 프로젝트 특징 및 기여점

- 감성 분석과 요구사항 분석을 동시에 수행하는 이중 출력 구조 제안
- 한국어 사용자 리뷰 특성에 최적화된 전처리 및 임베딩 설계
- 불균형 데이터 문제를 고려한 실험 설계
- 실제 서비스 적용을 고려한 분석 흐름 제시

---

## 🚀 향후 확장 방향

- Transformer 기반 모델(BERT, KoBERT) 적용
- 다양한 도메인(여행, 커머스, 모빌리티) 데이터로 일반화 검증
- 이미지·음성 리뷰를 포함한 멀티모달 분석
- 실제 운영 시스템 연계 및 자동화 분석 서비스 구현
