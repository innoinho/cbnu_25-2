# 📘 온라인 사용자 리뷰 기반 감성·요구사항 통합 분석 프로젝트
### (Sentiment & Requirement Joint Analysis Project)

본 저장소는 온라인 사용자 리뷰 데이터를 활용하여  
감성(Sentiment)과 요구사항(Requirement)을 통합적으로 분석하기 위한  
딥러닝 기반 모델을 단계적으로 설계·구현·확장한 프로젝트를 담고 있다.

본 프로젝트는 베이스라인 모델 → 고도화 모델 → 논문 및 수업 프로젝트 적용의 흐름으로 구성되며,  
각 파일은 동일한 연구 문제를 서로 다른 방법론과 실험 목적에 따라 구현한 결과물이다.

---

## 🎯 프로젝트 목표

- 사용자 리뷰 텍스트로부터
  - 감성 표현(긍·부정·중립)
  - 서비스 개선과 관련된 요구사항
  을 동시에 분석
- 감성 결과를 요구사항 분석에 반영하는 통합 학습 전략 검증
- 한국어 리뷰 데이터에 적합한 전처리·모델링 방식 비교
- 학위논문 및 수업 프로젝트로 활용 가능한 재현성 있는 실험 구조 설계

---

## 🧩 프로젝트 구성 개요

본 저장소의 파일들은 다음과 같은 역할을 가진다.

### 1️⃣ LSTM 기반 베이스라인 모델
- 형태소 분석(Mecab) 기반 전처리
- LSTM 공통 은닉층 + 이중 출력(Dual-Output) 구조
- 부정 리뷰에 대해 요구사항 학습을 강화하는  
  Anomaly-Weighted Sample Weight 전략 적용
- 요구사항 키워드 해석을 위한 YAKE 기반 분석 포함

→ 경량 모델 기반 비교·검증용 구현

---

### 2️⃣ KoBERT 기반 확장 모델
- 사전학습 언어모델(KoBERT)을 공통 인코더로 사용
- Transformer 기반 문맥 표현을 활용한 이중 출력 구조
- 감성·요구사항 동시 예측을 통한 성능 향상 검증

→ 고성능 모델 및 실서비스 적용 가능성 검토용 구현

---

### 3️⃣ 노트북 기반 실험 및 보고서 파일
- 딥러닝 심화 수업 프로젝트 결과 정리
- 졸업논문 실험 과정(V0.1, V0.2 등) 단계별 기록
- 실험 설정, 결과, 해석 과정을 재현 가능하게 정리

→ 연구·학습 과정 문서화 및 버전 관리 목적

---

## ⚙️ 공통 처리 흐름

1. 리뷰 데이터 수집 및 정제
2. 감성 라벨링 (평점 기반)
3. 요구사항 라벨링 (카테고리 기반)
4. 텍스트 전처리 및 벡터화
5. 이중 출력 모델 학습
6. 감성·요구사항 성능 평가 및 비교
7. 부정 리뷰 중심 요구사항 해석

---

## 🧠 핵심 아이디어

- 감성과 요구사항은 독립적인 문제가 아닌 상호 연관된 정보
- 부정 감성 리뷰는 서비스 개선 요구가 포함될 가능성이 높음
- 감성 결과를 활용한 요구사항 학습 가중 전략을 통해
  실질적인 사용자 불만 탐지 성능을 강화하고자 함

---

## 📊 활용 분야

- 온라인 서비스 리뷰 분석
- 고객 불만 자동 탐지 시스템
- 서비스 개선 인사이트 도출
- 학위논문 및 캡스톤 프로젝트
- 자연어 처리 기반 멀티태스크 학습 연구

---

## 📌 정리

본 프로젝트는 하나의 최종 모델만을 제시하기보다,  
문제 정의 → 베이스라인 → 고도화 → 비교·해석의 전체 연구 흐름을 담는 것을 목표로 한다.

각 파일은 서로 다른 실험 목적을 가지지만,  
모두 “온라인 리뷰에서 감성과 요구사항을 어떻게 함께 이해할 것인가”라는  
공통된 연구 질문을 중심으로 구성되어 있다.
